Никола любит классифицировать все вещи. Он классифицировал ряд чисел и в результате его усилий, простая последовательность чисел стала глубоко вложенным списком. София и Стефан не понимают как он организовал числа и нужно выяснить, что все это значит. Им нужна ваша помощь, чтобы понять сумасшедший список Николы.

Существует итератор списка, который содержит целые числа или другие вложенные итераторы списков, которые могут содержать еще несколько итераторов списков и целых чисел, которые затем... и так далее. Вы должны положить все целые значения, в один плоский список. Порядок должен быть такой же как и в первоначальном списке с представлением строки слева направо.
После чего ваша функция должна вернуть объект-итератор, ссылающийся на полученный список.

Мы должны скрыть эту программу от Николы, сохранив её маленькой и легкой. Из-за этого, Ваш код должен быть короче, чем 140 символов (с пробелами) .

Входные данные: Вложенный итератор с целыми числами и другими итераторами.

Выходные данные: Объект-итератор, ссылающийся на одномерный список с целыми числами.

Пример:

list(flat_list(iter([1, 2, 3]))) == [1, 2, 3]
list(flat_list(iter([1, iter([2, 2, 2]), 4]))) == [
                     1, 2, 2, 2, 4]
list(flat_list(iter([iter([2]), iter([4, iter([5, 6, iter([6]), 6, 6, 6]), 7])]))) == [
                     2, 4, 5, 6, 6, 6, 6, 6, 7]
list(flat_list(iter([-1, iter([1, iter([-2]), 1]), -1]))) == [
                     -1, 1, -2, 1, -1]

Как это используется: Эта концепция полезна для разбора и анализа файлов со сложной структурой и она бросает вызов вашей креативности в написании короткого кода.

Предусловия: 0 ≤ |array| ≤ 100
∀ x ∈ array : -232 < x < 232 or x is a list
depth < 10
